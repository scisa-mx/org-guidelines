# Guía de Configuración e Instalación para Montar el Entorno de Desarrollo

Esta guía proporciona una visión estructurada de las herramientas, SDKs y aplicaciones esenciales para configurar una estación de trabajo orientada al desarrollo de software en el ecosistema de Microsoft. Está optimizada para cubrir tanto los flujos de trabajo heredados (*legacy*) como la adopción tecnológica hacia **.NET 10** y **Visual Studio 2026**.

---

## 1. Entornos de Desarrollo Integrados (IDEs) y Editores

### Visual Studio (2026 y 2022)
Es el entorno de desarrollo integrado (IDE) principal para la plataforma .NET. Para este ecosistema se plantea una instalación y convivencia en paralelo (*side-by-side*):
* **Visual Studio 2026:** Optimizado para la creación de nuevos proyectos modernos aprovechando las capacidades completas de **.NET 10**.
* **Visual Studio 2022:** Mantenido estrictamente para dar soporte, compilar y realizar mantenimiento en proyectos basados en arquitecturas tradicionales o código heredado (*legacy code*).

### Visual Studio Code
Un editor de código fuente ligero, rápido y multiplataforma. Es la herramienta ideal para desarrollo frontend, tareas de scripting, automatización, edición de archivos de configuración (JSON, YAML, Markdown) y flujos de trabajo ágiles mediante su extenso ecosistema de extensiones.

---

## 2. Frameworks, SDKs y Entornos de Ejecución

### Microsoft .NET SDK (8.0.x, 9.0.x y 10.0.x)
El Kit de Desarrollo de Software (SDK) que incluye los compiladores, las bibliotecas base y la herramienta de línea de comandos (`dotnet CLI`). Son indispensables para compilar, probar y empaquetar aplicaciones. 
* Se deben instalar los SDKs correspondientes a **.NET 8** y **.NET 9** para dar soporte a las aplicaciones que ya se encuentran en producción o en etapas avanzadas de desarrollo.
* Se añade el **SDK de .NET 10.0.x** como requisito fundamental para el desarrollo de nuevas características nativas bajo la última versión de la plataforma.
*(Recuerda que los últimos dígitos del SDK indican la banda de características y el parche de seguridad; cualquier versión superior dentro de la misma versión mayor es completamente válida).*

### Microsoft .NET Framework 4.8
La última versión de la línea clásica e histórica de .NET exclusiva para el sistema operativo Windows. Su presencia es obligatoria para poder ejecutar, depurar y dar mantenimiento a proyectos *legacy* existentes que aún no han sido migrados a las versiones modernas y multiplataforma de .NET.

### Node.js
Entorno de ejecución para JavaScript en el lado del servidor. En un entorno mayoritariamente .NET, se utiliza principalmente para gestionar paquetes de dependencias frontend (`npm`), ejecutar herramientas de compilación web (como Webpack, Vite) y automatizar flujos de tareas.

### PostSharp Metalama
Un framework moderno de Programación Orientada a Aspectos (AOP) diseñado para .NET. Permite interceptar el código durante la compilación para inyectar lógica repetitiva (*boilerplate code*), como logging, validaciones o manejo de excepciones, manteniendo el código fuente excepcionalmente limpio.

## 3. Gestión de Bases de Datos

### Microsoft SQL Server
El motor de base de datos relacional (RDBMS) empresarial estándar para el ecosistema Microsoft. Es el servicio encargado del almacenamiento seguro, procesamiento transaccional y persistencia general de la información de las aplicaciones.

### SQL Server Management Studio (SSMS)
La herramienta de administración con interfaz gráfica (GUI) oficial para interactuar con bases de datos SQL Server. Permite a los desarrolladores escribir consultas complejas, diseñar esquemas de bases de datos, gestionar la seguridad del motor y optimizar el rendimiento de los índices y procedimientos almacenados.

## 4. Componentes del Sistema e Infraestructura de Hospedaje

### Microsoft .NET Windows Server Hosting (8.0.x, 9.0.x y 10.0.x)
Paquete de infraestructura diseñado para servidores web (o entornos locales de simulación y pruebas en IIS). Incluye el *runtime* de .NET y el módulo ASP.NET Core necesario para que el servidor web Internet Information Services (IIS) pueda alojar y ejecutar aplicaciones web.
* Al igual que con los SDKs, se deben tener las versiones correspondientes a **.NET 8** y **.NET 9**.
* Es indispensable incorporar el componente de **.NET 10.0.x Windows Server Hosting** en el entorno local si se planea probar el despliegue de aplicaciones locales sobre IIS utilizando la nueva versión.

### Microsoft Visual C++ 2015-2022 Redistributable (x64 y x86)
Bibliotecas en tiempo de ejecución necesarias para aplicaciones desarrolladas con herramientas de Visual C++. Son un prerrequisito crítico a nivel de sistema operativo, ya que múltiples componentes de software, bases de datos e IDEs dependen de estas librerías nativas para funcionar correctamente.

## 5. Herramientas de Desarrollo y Gestión

### Docker Desktop
Plataforma de contenerización que permite empaquetar una aplicación junto con todo su entorno y dependencias dentro de un contenedor aislado. Garantiza que el software funcione de manera idéntica y predecible desde la máquina local del desarrollador hasta los servidores de producción.

### Git
El sistema de control de versiones distribuido estándar de la industria. Es una herramienta esencial para realizar el seguimiento de los cambios en el código fuente, la gestión de ramas de trabajo (*branches*) y facilitar la colaboración segura entre múltiples desarrolladores.

### Postman
Una plataforma colaborativa y gráfica para el diseño, simulación y prueba de APIs (REST, GraphQL, etc.). Permite enviar peticiones HTTP personalizadas y analizar las respuestas devueltas por el backend sin necesidad de construir código de interfaz de usuario para las pruebas.

## 6. Autenticación y Acceso a Repositorios (SSH de Azure)

### Claves SSH para Azure DevOps / Azure Repos
Para poder interactuar con los repositorios de código de la organización (realizar clonación mediante `git clone`, enviar cambios con `git push` o descargar actualizaciones con `git pull`), es necesario configurar un mecanismo de autenticación seguro. 

El uso de **claves SSH (Secure Shell)** permite establecer una conexión cifrada y segura entre tu máquina local y los servidores de Azure DevOps sin necesidad de introducir tus credenciales institucionales o tokens de acceso (PAT) de forma manual en cada operación. Consiste en un par de llaves criptográficas: una llave privada que permanece segura en tu equipo y una llave pública que se registra en tu perfil de Azure.
