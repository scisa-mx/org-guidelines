# 3.2 Evaluación y Aprobación de Nuevas Dependencias

Después de evaluar el uso de la herramienta...

> Presenta a tu líder de proyecto un documento en `docs/adr/` con la siguiente información. *ADR → Architecture Decision Record*

- Problema que se quiere resolver.

- Alternativas evaluadas (incluyendo "hacerlo in-house").

- Librería seleccionada y por qué.

- Riesgos y plan de salida.

> OJO: El formato oficial para la documentación es Markdown

> Agrega la resolución del uso de librería en el documento con el razonamiento detrás de la decisión.
