# Configuracion oficial del IDE

Actualmente tenemos algunas configuraciones organizacionales para el IDE `Visual Studio`, todos los lineamientos se basan en tener este editorconfig con sus pautas para el lint de Visual Studio.

Puedes encontrar este archivo en [.editorconfig](https://github.com/scisa-mx/org-guidelines/blob/main/.editorconfig)

## Visual Studio 
### Configurar .editorconfig para todo el equipo

Visual Studio detecta automáticamente los archivos .editorconfig si están en el repositorio.

**Ubicación recomendada**
En la raíz del repositorio

```
repo/
 ├─ .editorconfig
 ├─ src/
 ├─ tests/
 └─ solution.sln
 ```

### Habilitar .editorconfig en Visual Studio

En Visual Studio:
```
Tools
 → Options
 → Text Editor
 → Code Style
 → General
 ```

Activar: 

```
✔ Enable .editorconfig support
✔ Prefer .editorconfig over IDE settings
```

### Estructura recomendada organizacional

En equipos suele verse así:
```
repo/
│
├─ .editorconfig
├─ .vsconfig
├─ .gitattributes
│
├─ src/
│   ├─ Api
│   ├─ Services
│   └─ Domain
│
├─ tests/
│
└─ solution.sln
```

## Visual Studio Code

No tenemos (hasta el momento de escribir este articulo) un documento o guías de lineamiento directos para proyectos que suelen utilizar solo Visual Studio Code, pero se recomienda (para proyectos de front-end para vue) las siguientes librerias:

- Git graph: `mhutchie.git-graph`
- Icons: `tal7aouy.icons`
- Priettier ESLint: `rvest.vs-code-prettier-eslint`
- Vue (Official): `Vue.volar`
- Vue 3 Snippets: `hollowtree.vue-snippets`
- Tailwind CSS IntelliSense: `bradlc.vscode-tailwindcss`
- Simply Markdown Viewer: `MarkdownViewer.enhanced-md-editor`

Solo con seguir las especificaciones de uso de cada Plugin deberia de ser suficiente para trabajar de forma emficiente con el stack de front end pensado en microservicios.
